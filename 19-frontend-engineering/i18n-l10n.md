# 🌍 i18n & l10n — আন্তর্জাতিকীকরণ ও স্থানীয়করণ

## 📌 সংজ্ঞা

```
  i18n  =  internationalization   = i + ১৮টি অক্ষর + n
  l10n  =  localization            = l + ১০টি অক্ষর + n
  g11n  =  globalization           = g + ১১টি অক্ষর + n
```

**i18n (internationalization)** — আপনার অ্যাপকে এমনভাবে design করা যাতে সহজে বিভিন্ন
ভাষা ও region-এ adapt করা যায়। developer-এর কাজ।

**l10n (localization)** — নির্দিষ্ট locale-এর জন্য content adapt করা — ভাষা translate,
date/number/currency format, image, color, regulation। translator/local team-এর কাজ।

**g11n (globalization)** — ব্যবসায়িক strategy: i18n + l10n + cultural + legal +
infrastructure।

```
  i18n হলো ছাঁচ                  l10n হলো কনটেন্ট
  ────────────                    ──────────────
  Date format: locale-aware       bn-BD: ২৭ অক্টোবর ২০২৪
  Number: locale-aware            ar-SA: ٢٧ أكتوبر ٢٠٢٤
  Translation system              en-US: October 27, 2024
  RTL support
  Pluralization framework
```

বাংলাদেশে গুরুত্বপূর্ণ:
- Daraz Bangla + English
- bKash বাংলা ও ইংরেজি
- 10 Minute School — ১৫০+ দেশের student, ১৬+ language

## 🏗️ i18n ভিত্তি

### ১. সব text externalize করুন

```jsx
// ❌ hard-coded
<h1>Welcome to Daraz</h1>

// ✅ key দিয়ে lookup
<h1>{t('home.welcome')}</h1>
```

```json
// en.json
{ "home.welcome": "Welcome to Daraz" }

// bn.json
{ "home.welcome": "ডারাজে স্বাগতম" }
```

### ২. String concatenation এড়ান

```javascript
// ❌ কখনো এমন করবেন না
const msg = 'Hello, ' + name + '! You have ' + n + ' messages.';

// বাংলায়: "হ্যালো, আকাশ! আপনার ৫টি বার্তা আছে।"
// আরবিতে word order ভিন্ন:
//   "مرحبا، آكاش! لديك ٥ رسائل."
// translator string পুনর্বিন্যাস করতে পারে না — এটাই issue
```

```javascript
// ✅ placeholder-based template
t('greeting', { name, count: n });
// "Hello, {name}! You have {count} messages."
```

### ৩. ICU MessageFormat — gold standard

ICU (International Components for Unicode) message format — pluralization, gender,
selection একসাথে handle করে।

```
{count, plural,
  =0    {You have no messages}
  one   {You have one message}
  other {You have # messages}
}
```

```
{gender, select,
  male   {He liked your photo}
  female {She liked your photo}
  other  {They liked your photo}
}
```

বাংলায় plural form ২টা: singular ও plural; আরবিতে ৬টা; রুশ-এ ৩টা — ICU সব সামলায়।

## 🇧🇩 Bangla (bn-BD) — বিশেষ বিবেচনা

### ১. Digit shaping

বাংলায় ০-৯ আলাদা glyph (০, ১, ২, ৩, ৪, ৫, ৬, ৭, ৮, ৯)। সিদ্ধান্ত নিতে হবে:
- Western digit রাখব (5,000 টাকা)?
- Bangla digit ব্যবহার করব (৫,০০০ টাকা)?

bKash, Nagad, Pathao, Daraz প্রায় সবাই **Bangla digit** ব্যবহার করে — UI বাংলা হলে।

```javascript
// Intl.NumberFormat দিয়ে auto
new Intl.NumberFormat('bn-BD').format(5000);  // "৫,০০০"
new Intl.NumberFormat('bn-BD-u-nu-latn').format(5000); // "5,000"  (Latin digit)

// Manual conversion
const banglaDigits = '০১২৩৪৫৬৭৮৯';
const toBangla = (num) =>
  String(num).replace(/\d/g, d => banglaDigits[+d]);

toBangla(5000);   // "৫০০০"
toBangla('01712345678');  // "০১৭১২৩৪৫৬৭৮"
```

⚠️ Phone number, OTP, transaction reference — **কখনো convert করবেন না**, copy-paste-এ
সমস্যা হবে।

### ২. Currency

```javascript
new Intl.NumberFormat('bn-BD', {
  style: 'currency',
  currency: 'BDT'
}).format(5000.50);
// "৫,০০০.৫০৳"

new Intl.NumberFormat('bn-BD', {
  style: 'currency',
  currency: 'BDT',
  maximumFractionDigits: 0
}).format(5000);
// "৫,০০০৳"

// লাখ/কোটি format (BD-specific)
function formatBdtBangla(amount) {
  if (amount >= 10_000_000) return (amount / 10_000_000).toFixed(2) + ' কোটি';
  if (amount >= 100_000)    return (amount / 100_000).toFixed(2) + ' লক্ষ';
  if (amount >= 1000)       return (amount / 1000).toFixed(1) + ' হাজার';
  return amount.toString();
}
formatBdtBangla(15_50_000); // "১৫.৫০ লক্ষ"
```

### ৩. Date

```javascript
const d = new Date('2024-10-27');

new Intl.DateTimeFormat('bn-BD', {
  dateStyle: 'long'
}).format(d);
// "২৭ অক্টোবর, ২০২৪"

new Intl.DateTimeFormat('bn-BD', {
  weekday: 'long',
  day: 'numeric',
  month: 'long',
  year: 'numeric'
}).format(d);
// "রবিবার, ২৭ অক্টোবর, ২০২৪"
```

### ৪. Font fallback

বাংলা proper render-এর জন্য আলাদা font লাগে:

```css
:root {
  --font-bangla: 'Noto Sans Bengali', 'Hind Siliguri', 'Kalpurush',
                 'SolaimanLipi', 'Siyam Rupali', sans-serif;
  --font-latin: 'Inter', 'SF Pro Display', system-ui, sans-serif;
  
  --font-stack: var(--font-latin), var(--font-bangla);
}

html[lang="bn-BD"] {
  --font-stack: var(--font-bangla), var(--font-latin);
}

body { font-family: var(--font-stack); }
```

### বাংলা font বাছাই

| Font | License | আরো |
|------|---------|-----|
| **Noto Sans Bengali** (Google) | Free, OFL | সবচেয়ে recommended, modern |
| **Hind Siliguri** (Google) | Free, OFL | clean UI |
| **Kalpurush** | Free | classic, প্রিন্ট-feel |
| **SolaimanLipi** | Free | Linux/Bangladesh standard |
| **Siyam Rupali** | Free | newspaper-feel |

**Self-host Google Font:**

```css
@font-face {
  font-family: 'Noto Sans Bengali';
  src: url('/fonts/noto-bengali-400.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;  /* FOUT > invisible text */
  unicode-range: U+0980-09FF;  /* শুধু Bengali range, Latin আলাদা font load করবে */
}
```

`unicode-range` দিয়ে browser শুধু সেই font load করবে যেখানে Bengali character আছে —
performance gain।

### ৫. Bangla typography

```css
[lang="bn-BD"] {
  line-height: 1.7;       /* English-এর তুলনায় বাংলা উচ্চ line-height চায় */
  letter-spacing: 0;       /* বাংলা conjunct ভেঙে যাবে letter-spacing-এ */
  word-spacing: 0.05em;
}
```

`letter-spacing` zero রাখুন — যুক্তবর্ণ (ক্ষ, ঞ্জ) ভাঙে।

```html

<!-- বাংলা search -->
```

## 🔄 RTL Support

আরবি, হিব্রু, ফারসি, উর্দু RTL (right-to-left)। বাংলা LTR, কিন্তু Daraz Group-এ Lazada
Pakistan (উর্দু) আছে।

```html
<html dir="rtl" lang="ar-SA">
```

### CSS Logical Properties

```css
/* ❌ পুরনো — RTL-এ ভুল হবে */
.card {
  margin-left: 16px;
  padding-right: 8px;
  border-left: 1px solid;
  text-align: left;
}

/* ✅ logical — auto-flip RTL-এ */
.card {
  margin-inline-start: 16px;    /* LTR-এ left, RTL-এ right */
  padding-inline-end: 8px;
  border-inline-start: 1px solid;
  text-align: start;
}
```

### Direction-aware components

```jsx
// Tailwind RTL plugin
<div className="ms-4 pe-2 text-start">  {/* margin-start, padding-end */}
```

```jsx
// Conditional flip
<Icon className={isRTL ? 'rotate-180' : ''} />
// arrow → এবং ← swap
```

## 📚 Library Selection

### i18next (framework-agnostic, recommended)

```bash
npm install i18next react-i18next i18next-http-backend
```

```javascript
// i18n.js
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import HttpBackend from 'i18next-http-backend';

i18n
  .use(HttpBackend)
  .use(initReactI18next)
  .init({
    fallbackLng: 'en',
    supportedLngs: ['en', 'bn-BD', 'ur-PK'],
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json'
    },
    ns: ['common', 'checkout', 'product'],
    defaultNS: 'common',
    interpolation: { escapeValue: false }, // React already escapes
    detection: {
      order: ['cookie', 'navigator'],
      caches: ['cookie']
    }
  });
```

```jsx
// Component
import { useTranslation } from 'react-i18next';

function Greeting({ name, count }) {
  const { t, i18n } = useTranslation();
  return (
    <>
      <h1>{t('home.welcome', { name })}</h1>
      <p>{t('messages.count', { count })}</p>
      <button onClick={() => i18n.changeLanguage('bn-BD')}>
        বাংলা
      </button>
    </>
  );
}
```

```json
// locales/bn-BD/common.json
{
  "home.welcome": "{{name}}, ডারাজে স্বাগতম",
  "messages.count_one": "আপনার ১টি বার্তা আছে",
  "messages.count_other": "আপনার {{count}}টি বার্তা আছে"
}
```

i18next plural suffix: `_zero`, `_one`, `_two`, `_few`, `_many`, `_other` (CLDR)।

### react-intl (FormatJS) — ICU MessageFormat native

```jsx
import { IntlProvider, FormattedMessage, FormattedNumber, FormattedDate } from 'react-intl';

<IntlProvider locale="bn-BD" messages={messages}>
  <FormattedMessage
    id="cart.items"
    defaultMessage="{count, plural, =0 {Cart empty} one {1 item} other {# items}}"
    values={{ count }}
  />
  
  <FormattedNumber value={5000} style="currency" currency="BDT" />
  <FormattedDate value={new Date()} dateStyle="long" />
</IntlProvider>
```

### Comparison

| | i18next | react-intl |
|--|---------|-----------|
| Bundle | ~৪০KB | ~৬০KB |
| Backend loader | rich plugins | manual |
| ICU support | via plugin | native |
| Server (Node) | ✅ | ✅ |
| Vue/Angular | ✅ | React-only |

বেশিরভাগ team i18next ব্যবহার করে, FormatJS more "correct" তবে heavy।

## 🚀 Lazy-loading translations

সব ভাষার সব translation একসাথে ship করলে bundle বিশাল।

```javascript
// i18next-http-backend automatically lazy loads
i18n.init({
  partialBundledLanguages: true,
  ns: ['common'],  // initial
  defaultNS: 'common'
});

// Component-এ namespace lazy load
const { t } = useTranslation('checkout');  // /locales/bn-BD/checkout.json fetch
```

```javascript
// Webpack/Vite dynamic import alternative
async function loadLocale(lang) {
  const messages = await import(`./locales/${lang}.json`);
  i18n.addResourceBundle(lang, 'common', messages.default);
}
```

ফলাফল: প্রথমে ১০-২০KB initial locale, route বদলালে অন্য namespace lazy load।

## 🌐 URL Strategy

```
  ১. Subdirectory (recommended)
     daraz.com/bn/products/...
     daraz.com/en/products/...
     ✅ SEO ভালো (single domain)
     ✅ shared session
  
  ২. Subdomain
     bn.daraz.com.bd, en.daraz.com.bd
     ⚠️ session sharing config লাগবে
  
  ৩. ccTLD per language (rare)
     daraz.bd vs daraz.bd-en
     ❌ expensive, complex
  
  ৪. Query param
     daraz.com.bd?lang=bn
     ❌ SEO worst, avoid
```

### Next.js i18n routing

```javascript
// next.config.js
module.exports = {
  i18n: {
    locales: ['en', 'bn-BD'],
    defaultLocale: 'en',
    localeDetection: true
  }
};
```

`/products/iphone-15` → English; `/bn-BD/products/iphone-15` → Bangla।

### Tools

```
  Crowdin     →  popular, GitHub integration, glossary, screenshot context
  Lokalise    →  developer-friendly, API/CLI, in-context editor
  Phrase      →  enterprise, ICU support, branching
```

### Workflow

```
  ┌────────────────────────────────────────────────────────────┐
  │                                                            │
  │  Developer        Translator                Production     │
  │  ─────────         ──────────                ──────────    │
  │  ① add string                                              │
  │     in en.json                                             │
  │  ② push to repo  ──▶ Crowdin sync                          │
  │                       └─▶ translator works                │
  │                       └─▶ review                          │
  │                       └─▶ approve                         │
  │  ③ Crowdin PR ◀───────                                    │
  │  ④ merge ────────────▶ deploy                             │
  │                                                            │
  └────────────────────────────────────────────────────────────┘
```

### Best practices for translators

- প্রতিটি key-এ context comment ("Button on checkout page")
- Screenshot attach (Crowdin auto)
- Glossary maintain ("cart" সবসময় "কার্ট" না "ঝুড়ি")
- ICU plural format শেখা team-কে

bKash অ্যাপ বাংলা ও ইংরেজি; OTP/PIN inputs digit-aware; transaction history bangla

```jsx
function TransactionItem({ tx }) {
  const { i18n } = useTranslation();
  const locale = i18n.language;
  
  return (
    <article>
      <h3>{t(`tx.type.${tx.type}`)}</h3>
      <time>
        {new Intl.DateTimeFormat(locale, {
          dateStyle: 'medium',
          timeStyle: 'short'
        }).format(tx.createdAt)}

      <p>
        {new Intl.NumberFormat(locale, {
          style: 'currency',
          currency: 'BDT',
          maximumFractionDigits: 2

      </p>
      <small>{t('tx.fee', {
        fee: new Intl.NumberFormat(locale, {
          style: 'currency', currency: 'BDT'
        }).format(tx.fee)
      })}</small>
    </article>
  );
}
```

```json
// bn-BD
{
  "tx.type.send": "টাকা পাঠালেন",
  "tx.type.receive": "টাকা পেলেন",
  "tx.type.cashout": "ক্যাশ আউট",
```

## ⚠️ Pitfalls

```
  ❌ String concatenation — translator word order বদলাতে পারে না
  ❌ Plurals if/else manual — language complexity miss (Russian 3 forms, Arabic 6)
  ❌ Hard-coded date format ("MM/DD/YYYY")
  ❌ Hard-coded currency symbol ($)
  ❌ Image-এ text — translate impossible (SVG-এ text use করুন)
  ❌ Fixed-width container — translation longer/shorter হতে পারে (German 30%+)
  ❌ Right-aligned arrows in RTL — flip করতে ভুলে যাওয়া
  ❌ Color meaning ভিন্ন culture-এ ভিন্ন (red = "danger" western, "luck" Chinese)
  ❌ Cultural assumption (left-to-right form flow)
  ❌ Locale fallback ভুল (en-GB → en, না কি en-US?)
  ❌ Bangla digit transaction reference — copy-paste fail
  ❌ Latin font fallback বাংলা character-এ — tofu (□) দেখাবে
  ❌ Translation key in production code (i18next missingKey unhandled)
```

```
  i18n vs l10n
  ════════════════════════════════════════════════
  i18n  →  preparation (developer)
  l10n  →  adaptation (translator + designer)
  
  Key building blocks
  ────────────────────
  ১. Externalize strings (translation file)
  ২. ICU MessageFormat (plural, gender, select)
  ৩. Intl API (number, date, currency, list, segment)
  ৪. RTL support (logical properties)
  ৫. Font subsetting (unicode-range)
  ৬. Lazy loading per locale
  ৭. URL strategy (subdir best for SEO)
  ৮. Translation workflow (Crowdin/Lokalise)
  
  বাংলাদেশের জন্য
  ─────────────────
  • bn-BD lang attribute
  • Noto Sans Bengali font stack
  • Bangla digit (০-৯) UI-তে, NOT identifiers
  • লাখ/কোটি format consideration
  • Crowdin/Lokalise বাংলা translator pool আছে
```

i18n shipping দিনে শুরু করতে হয়, পরে retrofit করতে হলে codebase-এর প্রতিটি কোণায় hard-coded
string খুঁজতে হয় — সেটা ভয়াবহ painful। বাংলাদেশের অ্যাপ-এ Bengali first-class citizen
হওয়া উচিত — Bengali typography, digit, font, plural — সব Latin-এর মতো care নিয়ে। সেটাই
ইউজারের ভাষাকে সম্মান।
