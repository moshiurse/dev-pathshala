বাংলাদেশের টেক ইন্ডাস্ট্রিতে একটা সাধারণ সমস্যা আছে — কোড লেখা হয়, কিন্তু কেন সেই

// প্রস্তাবিত স্কোরিং সার্ভিস — সরলীকৃত উদাহরণ

class RideMatchingService
{
    private const WEIGHT_DISTANCE    = 0.25;
    private const WEIGHT_ETA         = 0.35;
    private const WEIGHT_RATING      = 0.20;
    private const WEIGHT_DESTINATION = 0.20;

    /**
     * নিকটবর্তী ড্রাইভারদের মধ্যে সেরাটি নির্বাচন করুন
     *
     * @param  RideRequest  $request   রাইডারের রিকোয়েস্ট
     * @param  Driver[]     $drivers   ২ কি.মি. রেডিয়াসের ড্রাইভার
     * @return Driver[]                স্কোর অনুযায়ী সর্টেড টপ ৩
     */
    public function rankDrivers(RideRequest $request, array $drivers): array
    {
        $scored = [];

        foreach ($drivers as $driver) {
            $distScore = $this->normalizeDistance(
                $driver->distanceTo($request->pickupLocation)
            );
            $etaScore = $this->normalizeETA(
                $this->trafficService->estimateETA($driver, $request)
            );
            $ratingScore   = $driver->rating / 5.0;
            $destScore     = $this->destinationMatch($driver, $request);

            $totalScore = (self::WEIGHT_DISTANCE * $distScore)
                        + (self::WEIGHT_ETA * $etaScore)
                        + (self::WEIGHT_RATING * $ratingScore)
                        + (self::WEIGHT_DESTINATION * $destScore);

            $scored[] = ['driver' => $driver, 'score' => $totalScore];
        }

        usort($scored, fn($a, $b) => $b['score'] <=> $a['score']);

        return array_slice(
            array_column($scored, 'driver'),
            0,
            3
        );
    }
}

/**
 * Pathao Food-এ অর্ডারের ডেলিভারি ফি ক্যালকুলেট করুন।
 *
 * ফি ক্যালকুলেশন লজিক:
 * - বেস ফি: ৩০ টাকা (৩ কি.মি. পর্যন্ত)
 * - অতিরিক্ত: প্রতি কি.মি. ১০ টাকা
 * - পিক আওয়ার (১২-২টা, ৭-৯টা): ১.৫x সার্চার্জ
 * - বৃষ্টির দিনে: ২x সার্চার্জ (WeatherAPI থেকে চেক)
 * - ন্যূনতম ফি: ৩০ টাকা, সর্বোচ্চ: ২৫০ টাকা
 *
 * @param {Object} order
 * @param {number} order.distanceKm        - রেস্টুরেন্ট থেকে কাস্টমার দূরত্ব (কি.মি.)
 * @param {string} order.restaurantArea    - রেস্টুরেন্টের এলাকা ('gulshan'|'dhanmondi'|...)
 * @param {Date}   order.orderTime         - অর্ডারের সময়
 * @param {boolean} [order.isPriority=false] - প্রাইওরিটি ডেলিভারি কিনা
 *
 * @returns {number} ডেলিভারি ফি টাকায় (পূর্ণ সংখ্যা, রাউন্ডেড)
 *
 * @example
 * calculateDeliveryFee({
 *   distanceKm: 5.2,
 *   restaurantArea: 'gulshan',
 *   orderTime: new Date('2024-11-11T13:00:00+06:00'),
 *   isPriority: false
 * });
 * // রিটার্ন: ৮২ (বেস ৩০ + অতিরিক্ত ২২ = ৫২, পিক আওয়ার ১.৫x = ৭৮, রাউন্ডেড ৮২)
 */
function calculateDeliveryFee(order) {
    const BASE_FEE = 30;
    const BASE_DISTANCE_KM = 3;
    const PER_KM_FEE = 10;
    const MIN_FEE = 30;
    const MAX_FEE = 250;

    let fee = BASE_FEE;
    if (order.distanceKm > BASE_DISTANCE_KM) {
        fee += (order.distanceKm - BASE_DISTANCE_KM) * PER_KM_FEE;
    }

    if (isPeakHour(order.orderTime)) {
        fee *= 1.5;
    }

    return Math.min(MAX_FEE, Math.max(MIN_FEE, Math.round(fee)));
}
```

## 📌 ৮. কখন ব্যবহার করবেন / করবেন না

### কখন RFC লিখবেন ✅

```
✅ RFC লিখুন যখন:
├── নতুন সার্ভিস/সিস্টেম তৈরি হচ্ছে
├── বড় আর্কিটেকচার পরিবর্তন (মনোলিথ → মাইক্রোসার্ভিস)
├── নতুন থার্ড-পার্টি ইন্টিগ্রেশন (পেমেন্ট গেটওয়ে, SMS প্রোভাইডার)
├── ডেটাবেস স্কিমায় বড় পরিবর্তন
└── ক্রস-টিম প্রভাব ফেলবে এমন কোনো কাজ

❌ RFC লিখবেন না যখন:
├── ছোট বাগ ফিক্স
├── UI টুইক বা কপি চেঞ্জ
├── রুটিন ডিপেন্ডেন্সি আপডেট
├── এক-দুই দিনের কাজ যা শুধু আপনার টিমে সীমাবদ্ধ
└── ইতোমধ্যে অনুমোদিত ডিজাইনের ইমপ্লিমেন্টেশন ডিটেইল

✅ পোস্টমর্টেম লিখুন যখন:

### বাংলাদেশি কোম্পানিতে ডকুমেন্টেশন কালচার তৈরির টিপস

```
  ১. ছোট থেকে শুরু করুন
  ──────────────────────
  প্রথমে শুধু ADR লেখা শুরু করুন — ১ পেজের ডক
  "আমরা কেন Redis বেছে নিলাম Memcached-এর বদলে"

  ২. টেমপ্লেট তৈরি করুন
  ────────────────────────
  RFC, পোস্টমর্টেম, ADR — প্রতিটির জন্য রেডিমেড টেমপ্লেট
  রাখুন। খালি ফাঁকা পেজ দেখলে কেউ লিখতে চাইবে না।

  ৩. কোড রিভিউতে ডক চেক করুন
  ──────────────────────────────
  PR রিভিউতে জিজ্ঞেস করুন: "এই চেঞ্জের জন্য কোনো ডক
  আপডেট দরকার?" — এটা হ্যাবিট তৈরি করবে।

  ৪. "ডক ডে" রাখুন
  ──────────────────
  মাসে একদিন পুরো টিম শুধু ডকুমেন্টেশন লিখবে/আপডেট
  করবে। Pathao, bKash-এর মতো কোম্পানিতে এটা কার্যকর।

  ৫. ভালো ডকের প্রশংসা করুন
  ──────────────────────────────
  যে ইঞ্জিনিয়ার ভালো RFC বা পোস্টমর্টেম লেখে, তাকে
  পাবলিকলি প্রশংসা করুন। এটা কালচার তৈরি করতে সাহায্য করে।
```
