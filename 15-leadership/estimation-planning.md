### 🏠 ২.১ T-Shirt Sizing (XS, S, M, L, XL)

T-Shirt Sizing হলো দ্রুত, উচ্চ-স্তরের এস্টিমেশনের জন্য সবচেয়ে সহজ পদ্ধতি।
এখানে সুনির্দিষ্ট সংখ্যা না দিয়ে আপেক্ষিক আকারে কাজের পরিমাণ বোঝানো হয়।
যখন ব্যাকলগে অনেক আইটেম আছে এবং দ্রুত সাজাতে হবে, তখন এটি কাজে আসে।

```

╠══════════════════════════════════════════════════════════════╣

║   XS ──► খুব ছোট কাজ     (কয়েক ঘণ্টা)                     ║
║    S ──► ছোট কাজ          (১-২ দিন)                         ║
```

- ❌ ক্লায়েন্টকে সুনির্দিষ্ট ডেডলাইন দিতে

### 🎯 ২.২ Story Points (Fibonacci: 1, 2, 3, 5, 8, 13, 21)

Story Points হলো আপেক্ষিক এস্টিমেশন পদ্ধতি যেখানে ফিবোনাচ্চি সিরিজ ব্যবহার করা
হয়। এর মূল ধারণা হলো — কাজের জটিলতা, পরিশ্রম এবং অনিশ্চয়তা একসাথে মিলিয়ে
একটি আপেক্ষিক সংখ্যা দেওয়া। ফিবোনাচ্চি সিরিজ ব্যবহারের কারণ হলো বড় সংখ্যার
মধ্যে পার্থক্য বাড়ে, যা বড় কাজের অনিশ্চয়তাকে প্রতিফলিত করে।

**JavaScript উদাহরণ — Story Point ট্র্যাকিং:**

```javascript
// Sprint velocity tracking — Grameenphone ডেভ টিম
const sprintData = [
    { sprint: 'Sprint 1', planned: 34, completed: 28 },
    { sprint: 'Sprint 2', planned: 30, completed: 31 },
    { sprint: 'Sprint 3', planned: 32, completed: 30 },
    { sprint: 'Sprint 4', planned: 31, completed: 33 },
];

function calculateVelocity(sprints) {
    const totalCompleted = sprints.reduce(
        (sum, s) => sum + s.completed, 0
    );
    return totalCompleted / sprints.length;
}

const avgVelocity = calculateVelocity(sprintData);
console.log(`Average Velocity: ${avgVelocity} points/sprint`);
// Output: Average Velocity: 30.5 points/sprint

// পরবর্তী স্প্রিন্টে ৩০-৩১ পয়েন্ট নেওয়া নিরাপদ
function suggestCapacity(velocity) {
    return {
        safe: Math.floor(velocity * 0.85),
        normal: Math.round(velocity),
        stretch: Math.ceil(velocity * 1.1),
    };
}

console.log(suggestCapacity(avgVelocity));
// { safe: 25, normal: 31, stretch: 34 }
```

#### কখন ব্যবহার করবেন:
- ✅ স্প্রিন্ট প্ল্যানিংয়ে — velocity দিয়ে ক্যাপাসিটি নির্ধারণে
- ✅ টিমের দীর্ঘমেয়াদী পারফরম্যান্স ট্র্যাকিংয়ে
- ✅ ব্যাকলগ আইটেম তুলনা ও প্রায়োরিটাইজেশনে

#### কখন ব্যবহার করবেন না:
- ❌ ক্লায়েন্টকে "কত দিন লাগবে" জানাতে (পয়েন্ট ≠ দিন)
- ❌ ইন্ডিভিজুয়াল ডেভেলপারের পারফরম্যান্স মাপতে
- ❌ ছোট, একক ব্যক্তির প্রজেক্টে

### 🕐 ২.৩ Time-Based Estimation (ঘণ্টা / দিন)

সময়-ভিত্তিক এস্টিমেশন সবচেয়ে সহজবোধ্য — কাজটি করতে কত ঘণ্টা বা দিন লাগবে
তা সরাসরি বলা হয়। ক্লায়েন্ট ও ম্যানেজমেন্ট এটি সবচেয়ে ভালো বোঝেন, তবে
এটিই সবচেয়ে বিপজ্জনক কারণ সময় কমিটমেন্ট হিসেবে ধরে নেওয়া হয়।

```
╔═══════════════════════════════════════════════════════════════╗
║           Time-Based Estimation পদ্ধতি                        ║
╠═══════════════════════════════════════════════════════════════╣
║                                                               ║
║   Three-Point Estimation (PERT):                              ║
║                                                               ║
║   Best Case (O) ───── সবকিছু ঠিকঠাক গেলে                    ║
║   Most Likely (M) ─── বাস্তবসম্মত ধারণা                      ║
║   Worst Case (P) ──── সমস্যা হলে                              ║
║                                                               ║
║   সূত্র: E = (O + 4M + P) / 6                                ║
```

**PHP উদাহরণ — PERT ক্যালকুলেটর:**

```php
<?php

class PERTEstimator
{
    /**
     * PERT (Program Evaluation and Review Technique) এস্টিমেশন
     * Daraz ডেভ টিমের জন্য টাস্ক এস্টিমেশন টুল
     */
    public function calculate(
        float $optimistic,
        float $mostLikely,
        float $pessimistic
    ): array {
        $expected = ($optimistic + 4 * $mostLikely + $pessimistic) / 6;
        $stdDev = ($pessimistic - $optimistic) / 6;

        return [
            'expected'       => round($expected, 1),
            'std_deviation'  => round($stdDev, 1),
            'confidence_68'  => [
                round($expected - $stdDev, 1),
                round($expected + $stdDev, 1),
            ],
            'confidence_95'  => [
                round($expected - 2 * $stdDev, 1),
                round($expected + 2 * $stdDev, 1),
            ],
        ];
    }
}

$estimator = new PERTEstimator();
$result = $estimator->calculate(
    optimistic: 3,
    mostLikely: 5,
    pessimistic: 10
);

print_r($result);
// expected: 5.5
// 68% confidence: 3.3 - 7.7 দিন
// 95% confidence: 1.2 - 9.8 দিন
```

```php
class TaskBreakdown
{
    private array $tasks = [];

    public function addEpic(string $name, array $subtasks): void
    {
        $this->tasks[$name] = [
            'subtasks'     => $subtasks,
            'total_points' => array_sum(
                array_column($subtasks, 'points')
            ),
        ];
    }

    public function getEstimate(string $epicName): array
    {
        $epic = $this->tasks[$epicName];
        $velocity = 30; // পয়েন্ট/স্প্রিন্ট

        return [
            'total_points'    => $epic['total_points'],
            'sprints_needed'  => ceil($epic['total_points'] / $velocity),
            'weeks_estimated' => ceil($epic['total_points'] / $velocity) * 2,
            'buffer_weeks'    => ceil(
                $epic['total_points'] / $velocity * 2 * 0.2
            ),
        ];
    }
}
```

```javascript
// Grameenphone ডেভ টিম — টেকনিক্যাল ডেট স্কোরকার্ড
const techDebtItems = [
    {
        area: 'Authentication Module',
        severity: 'high',
        estimatedEffort: 13, // story points to fix
        interestRate: 0.3,   // 30% extra effort on related features
        description: 'Legacy session-based auth, JWT তে মাইগ্রেট করতে হবে',
    },
    {
        area: 'Database Queries',
        severity: 'medium',
        estimatedEffort: 8,
        interestRate: 0.2,
        description: 'N+1 query সমস্যা, Eager loading দরকার',
    },
    {
        area: 'CSS Architecture',
        severity: 'low',
        estimatedEffort: 5,
        interestRate: 0.1,
        description: 'Inline styles সরিয়ে Tailwind CSS ব্যবহার করা',
    },
];

function calculateDebtImpact(items, sprintsAhead = 6) {
    return items.map(item => {
        const interestPerSprint = item.estimatedEffort * item.interestRate;
        const totalInterest = interestPerSprint * sprintsAhead;
        return {
            area: item.area,
            fixCost: item.estimatedEffort,
            interestOver6Sprints: Math.round(totalInterest),
            recommendation:
                totalInterest > item.estimatedEffort
                    ? '🔴 এখনই ঠিক করুন'
                    : '🟡 পরবর্তী কোয়ার্টারে',
        };
    });
}

console.table(calculateDebtImpact(techDebtItems));
// Authentication: fix=13, interest=23.4 → 🔴 এখনই ঠিক করুন
// Database:       fix=8,  interest=9.6  → 🔴 এখনই ঠিক করুন
// CSS:            fix=5,  interest=3.0  → 🟡 পরবর্তী কোয়ার্টারে
```
