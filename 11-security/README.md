# 🔐 সিকিউরিটি (Security)

> সফটওয়্যার সিকিউরিটির মূল ধারণা ও প্রয়োগ কৌশল

সফটওয়্যার ডেভেলপমেন্টে সিকিউরিটি একটি অপরিহার্য বিষয়। একজন সিনিয়র ইঞ্জিনিয়ার হিসেবে শুধু ফিচার তৈরি করা নয়, সেই ফিচারকে নিরাপদ রাখাও আপনার দায়িত্ব। এই সেকশনে আমরা সিকিউরিটির বিভিন্ন দিক নিয়ে আলোচনা করব — আক্রমণের ধরন, প্রতিরোধ কৌশল, এনক্রিপশন, অথেনটিকেশন এবং আরও অনেক কিছু।

## 🎯 কেন সিকিউরিটি শেখা জরুরি?

- **ডেটা সুরক্ষা:** ইউজারের ব্যক্তিগত তথ্য রক্ষা করা
- **ব্যবসায়িক ক্ষতি রোধ:** সিকিউরিটি ব্রিচ কোম্পানির সুনাম ও আর্থিক ক্ষতি করতে পারে
- **আইনি বাধ্যবাধকতা:** GDPR, PCI-DSS ইত্যাদি কমপ্লায়েন্স মেনে চলা
- **ক্যারিয়ার গ্রোথ:** সিনিয়র ইঞ্জিনিয়ারদের জন্য সিকিউরিটি জ্ঞান অপরিহার্য

## 🛡️ সিকিউরিটির মূলনীতি (CIA Triad)

```
        Confidentiality (গোপনীয়তা)
              /\
             /  \
            /    \
           /      \
          /________\
   Integrity      Availability
   (অখণ্ডতা)     (প্রাপ্যতা)
```

- **Confidentiality:** শুধু অনুমোদিত ব্যক্তিরাই ডেটা দেখতে পারবে
- **Integrity:** ডেটা অননুমোদিতভাবে পরিবর্তন করা যাবে না
- **Availability:** প্রয়োজনের সময় ডেটা ও সিস্টেম পাওয়া যাবে
